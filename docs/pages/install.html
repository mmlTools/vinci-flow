Requirements

  • OBS Studio installed and working.
  • vinci-flow plugin installed (copy to your OBS plugins folder or install via your distribution package).
  • obs-websocket is optional unless you plan to automate the plugin externally. (Vendor requests/events require obs-websocket v5 API.)

First-time setup checklist

Important
The plugin needs an Output Directory to write the generated overlay HTML files. If no output directory is configured, actions like “Create Lower Third” can fail with No output dir configured.
  1. Open the dock
    In OBS, enable the plugin dock (usually via View → Docks) and open VinciFlow.
  2. Set the Output Directory
    Choose a folder where the plugin will write overlay files (including lt-*.html and state/config files).
  3. Ensure the Browser Source exists
    The plugin is designed to manage a Browser Source named VinciFlow with a 1920×1080 canvas. If your workflow is manual, create a Browser Source and point it to the generated overlay file inside your Output Directory.
  4. Create your first lower third
    Use the dock action to create a default lower third, then edit title/subtitle and styling.
  5. Test show/hide
    Toggle visibility from the dock and confirm it appears in your scene.

Troubleshooting

“No output dir configured”

Set the Output Directory in the dock/settings first.

Nothing appears on stream

Verify the managed Browser Source is in your current scene and not hidden behind other sources. Confirm it is sized to 1920×1080 and not cropped.

Old design still showing

If you’re editing templates or assets, refresh the Browser Source (or disable/enable it) to force reload.

Animations not playing

If you use custom handling, ensure your theme provides root.__slt_show / root.__slt_hide and returns a Promise for hide.
